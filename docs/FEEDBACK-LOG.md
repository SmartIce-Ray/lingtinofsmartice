# 产品反馈与需求记录

> 集中记录 Lingtin 产品的**真实用户反馈**和**功能需求**，驱动产品持续改进。
>
> - 本文档只记录产品层面的反馈与需求，技术问题（bug、代码）通过 GitHub Issues 管理
> - 每次功能迭代后，同步更新对应条目状态

## 记录规范

### 编号体系

| 前缀 | 含义 | 示例 |
|------|------|------|
| `FB-` | 用户反馈（体验问题、使用障碍、正向评价） | FB-001 |
| `REQ-` | 功能需求（新功能请求、改进建议） | REQ-001 |

### 优先级定义

| 等级 | 含义 | 响应要求 |
|------|------|----------|
| **P0** | 阻断核心流程，用户无法完成关键操作 | 立即处理 |
| **P1** | 严重影响体验，但有替代路径 | 当前迭代处理 |
| **P2** | 改善体验，不影响核心流程 | 排入后续迭代 |
| **P3** | 锦上添花，长期优化方向 | 需求池积累 |

### 状态流转

```
待评估 → 已采纳 → 已排期 → 已上线
              ↘ 暂缓（附原因）
```

### 反馈来源标注

标明反馈者角色，便于理解用户视角：

- **店长** — 日常使用录音、看板、行动建议的一线用户
- **管理层** — 老板/区域经理，关注跨店数据和经营决策
- **店员** — 服务员/厨师，未来用户群体
- **顾客** — 终端消费者（间接反馈）
- **内部** — 团队成员在开发/测试中发现的产品问题
