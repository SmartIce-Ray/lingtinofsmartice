# AI 行动建议

## 为什么需要行动建议

收集顾客反馈只是开始，真正重要的是**采取行动**。AI 行动建议自动从当天的负面反馈中提炼出需要改进的问题，按优先级排列，并给出具体可执行的建议——帮你从「知道有问题」到「知道怎么改」。

## 前置条件

- 已登录店长账号
- 当天有已完成的桌访录音（尤其是包含负面反馈的录音）

## 在哪里找到行动建议

打开数据看板（`/dashboard`），滚动到页面底部的「AI 行动建议」卡片。

## 生成行动建议

### 首次生成

如果今天还没有生成过建议，卡片上方显示「生成今日建议」按钮。点击后：

1. AI 收集今天所有桌访中的负面反馈
2. 将相似的问题归类合并（如「太咸」和「偏咸」归为同一个问题）
3. 根据涉及的桌数确定优先级
4. 生成 3-5 条具体的改善建议

整个过程约 10-20 秒，期间按钮显示「生成中...」。

### 重新生成

如果已有建议但你又录了新的桌访，可以点击「重新生成」按钮。这会用最新的反馈数据重新分析，替换之前未处理的建议（已知悉或已解决的建议会保留）。

## 理解行动建议

每条建议包含以下信息：

### 优先级

| 优先级 | 颜色 | 含义 |
|--------|------|------|
| **高** | 红色 | 3 桌及以上提到同一问题，需要优先处理 |
| **中** | 黄色 | 2 桌提到同一问题，建议关注 |
| **低** | 蓝色 | 1 桌提到的问题，可择机处理 |

### 分类

| 分类 | 含义 | 示例 |
|------|------|------|
| 菜品质量 | 菜品口味、分量、卖相等 | 清蒸鲈鱼偏咸、红烧肉分量小 |
| 服务速度 | 上菜、结账等等候时间 | 午市高峰上菜慢 |
| 环境 | 温度、卫生、噪音等 | 空调温度偏高 |
| 员工态度 | 服务态度、专业度 | 服务员不够热情 |
| 其他 | 不属于以上类别的问题 | — |

### 建议内容

AI 生成的一段自然语言建议，说明问题是什么、涉及多少桌、建议怎么改进。例如：

> 3 桌顾客反映清蒸鲈鱼偏咸，建议与厨师长沟通减少盐量

### 原始反馈证据

每条建议都附带「查看原始反馈」链接，点击展开可以看到：
- 哪些桌提到了这个问题
- 顾客的原话是什么

这让你可以验证 AI 的归纳是否准确，也帮助你在和团队沟通时引用具体案例。

## 处理行动建议

每条建议有三种操作：

### 知悉

点击「知悉」表示你已经了解这个问题。建议状态变为「已知悉」（蓝色背景），但仍需后续处理。

### 标记已解决

点击「已解决」（待处理状态）或「标记已解决」（已知悉状态），会出现一个备注输入框。你可以：
- 直接点击「确认」——不写备注也可以
- 输入处理说明，如「与厨师长沟通，已减盐量」，再点击「确认」

解决后建议显示为绿色背景，备注内容也会显示在卡片上。

### 忽略

如果认为某条建议不需要处理（如偶发情况），点击「忽略」将其从列表中移除。被忽略的建议不会再显示，但数据库中保留记录。

## 推荐的处理流程

```
1. 午市结束后，先生成今日建议
2. 从高优先级开始看——3桌以上反映的问题最需要关注
3. 点击「查看原始反馈」确认问题确实存在
4. 能立即处理的，处理后标记「已解决」并写备注
5. 需要沟通协调的，先点「知悉」，处理完再回来标记
6. 明确是偶发情况的，可以「忽略」
7. 晚市前处理完毕，晚市结束后可以重新生成
```

## 常见问题

**Q: 没有生成任何建议？**
A: 说明今天的桌访中没有检测到负面反馈，这是好事。如果你确信有顾客表达了不满，可能是录音质量问题导致 AI 未能准确识别。

**Q: 建议不准确怎么办？**
A: 点击「查看原始反馈」核实。如果 AI 误解了顾客意思，直接忽略即可。AI 会随着数据积累不断改进。

**Q: 忽略的建议能找回吗？**
A: 被忽略的建议从界面上移除，但数据保留在系统中。如果需要回顾，可以通过 AI 智库查询历史数据。
