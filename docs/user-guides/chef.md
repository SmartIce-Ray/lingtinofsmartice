# 厨师长使用手册

## 你的角色

作为厨师长，你是后厨品质的最终把关人。Lingtin 把顾客对菜品的真实评价直接送到你面前——不用等店长口头转述，3 秒看到哪些菜需要改进、顾客原话怎么说的。

## 四个核心页面

| 页面 | 路径 | 一句话说明 |
|------|------|-----------|
| **AI 智库** | `/chef/chat` | 登录后首页——每日厨房汇报 + 菜品问题提问 + 行动链接 |
| **厨房待办** | `/chef/dashboard` | 与厨房相关的改善任务，知悉/解决/忽略 |
| **厨房反馈** | `/chef/dishes` | 全维度反馈（口味/速度/温度/卖相/食材）+ 对话溯源 + 播放录音 |
| **厨房会议** | `/chef/meetings` | 厨房内部会议录音，AI 自动生成纪要 |

## 典型一天的流程

```
──── 早间 ────
1. 打开 App → AI 智库自动推送每日厨房汇报（菜品差评、好评、待办提醒）
2. 点击汇报中的行动链接按钮，直接跳转到待办或菜品页

──── 午市前 ────
3. 切到「厨房待办」→ 查看昨日差评产生的改善任务
4. 重点关注置顶高亮的高频问题，安排当日改进

──── 午市/晚市后 ────
5. 切到「菜品反馈」→ 查看差评菜
6. 展开看顾客原话 → 不确定时点播放按钮听录音
7. 支持日期范围选择（今日/昨日/近7天）对比变化

──── 收餐后 ────
8. 参加复盘会，或在「厨房会议」查看 AI 会议纪要
9. 根据反馈调整次日备餐计划
```

## 厨房反馈（核心功能）

### 功能价值

- 差评菜品优先展示，3 秒定位问题
- 全维度覆盖：不只口味，还有出菜速度、温度、卖相、食材新鲜度
- 直接看顾客原话（"偏咸" ×2），不是二手转述
- 不确定时点播放按钮听录音，完全还原现场

### 使用场景

- **每餐结束后**：快速检查有没有菜出问题
- **收到差评时**：溯源具体是哪桌、顾客怎么说的
- **调整菜品前**：确认问题是普遍的还是偶发的

### 操作步骤

**1. 进入厨房反馈**
底部导航点「菜品」→ 页面分为上下两个区域。

**2. 查看需要关注的菜品**
上方「需要关注」区展示有差评的菜品，按差评数从多到少排列。每道菜显示：
- 菜名 + 差评桌数
- 反馈标签：如「偏咸 ×2」「不新鲜 ×1」，快速了解问题类型

**3. 展开查看对话详情**
点击菜品卡片展开，看到每桌的完整对话：
- 桌号 + 具体问题标签
- 店长的提问和顾客的回答原文

**4. 播放原声录音**
对话区点击播放按钮听原始录音。播放中再次点击暂停，同时只能播放一条录音。

**5. 执行行动**
每项问题可点「已改善」标记处理或「查看待办」跳转待办页。

**6. 切换日期**
页面顶部切换「今日/昨日」，对比菜品反馈变化。

**7. 查看好评菜品**
下方「表现不错」区展示零差评菜品，了解哪些菜受顾客欢迎。

### 常见问题

- **差评原话不准确？** → 点播放按钮听录音确认，AI 转写偶尔会有偏差。
- **没有菜品数据？** → 厨房反馈来源于店长的桌访录音，需要店长先完成当日桌访。
- **好评菜也想看详情？** → 目前好评区只显示汇总，详细对话后续支持。

## 厨房待办

### 功能价值

只看与厨房相关的待办（菜品质量 + 出菜速度等），不被服务、环境等无关建议干扰。聚焦你能改进的事情。

### 操作步骤

**1. 查看待处理任务**
底部导航点「待办」→ 显示与厨房相关的改善建议，按优先级排列。「建议优先处理」区域置顶高亮高频 + 高优先级问题。

**2. 处理任务**

| 操作 | 说明 |
|------|------|
| **知悉** | 已了解问题，标记为已知悉 |
| **已解决** | 问题已处理，可附备注（如"已调整盐量配比"） |
| **忽略** | 偶发情况不需处理 |

### 常见问题

- **待办从哪来？** → AI 分析桌访中的负面反馈后自动生成，也可能来自复盘会讨论。
- **没有待办？** → 说明近期没有菜品相关的差评，继续保持。

## 厨房会议

### 功能价值

厨房晨会、备餐讨论也能 AI 记录，自动生成纪要。不用手写会议记录，专注讨论本身。

### 操作步骤

**1. 进入会议页**
底部导航点「会议」。

**2. 录制会议**
选择「厨房会议」类型 → 点击录音按钮开始 → 讨论完毕后停止录音。

**3. 查看 AI 纪要**
处理完成后点击已完成的录音，查看：
- **会议摘要**：讨论内容概述
- **关键决定**：明确做出的决定
- **行动待办**：具体事项 + 负责人

### 常见问题

- **录音时长建议？** → 厨房会议一般 5-15 分钟，保持简短高效。
- **手机放哪里？** → 放在讨论区域中央，确保能收到所有人的声音。

## AI 智库（登录后首页）

### 每日厨房汇报

每天首次打开 App，AI 自动推送一份厨房视角的智能汇报，内容包括：
- 昨日菜品差评：具体问题 + 顾客原话 + 改进方向
- 好评菜品：保持当前做法
- 厨房待办提醒：数量和链接

汇报中的**行动链接**渲染为可点击的按钮（如「查看厨房待办 →」「查看菜品反馈 →」），点击直接跳转到对应页面。汇报末尾的**追问建议**显示为快捷 pill，点击即可继续提问。

> 每日汇报每天只推送一次。清空对话后会重新触发。

### 自由提问

除了每日汇报，你可以随时用自然语言向 AI 提问菜品相关问题：

| 场景 | 示例 |
|------|------|
| 菜品排查 | 酸菜鱼最近收到什么反馈？ |
| 差评分析 | 本周被差评最多的菜是哪道？ |
| 改进方向 | 花生不脆应该怎么调整？ |
| 趋势追踪 | 这周和上周相比差评有减少吗？ |

> AI 只能访问**本店厨房相关数据**。

## 提交产品反馈

遇到系统问题或有功能需求时，可以随时提交反馈。

**入口：** 右上角头像 → 「提交反馈」（路径：`/feedback`）。

**提交方式：**
- **文字模式**：输入反馈内容，可附带最多 3 张图片
- **语音模式**：录音后自动转写并分类

**查看我的反馈：** 头像 → 「我的反馈」（路径：`/feedback/history`），查看提交历史、处理状态和管理层回复。
