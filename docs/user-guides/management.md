# 管理层使用手册

## 你的角色

作为管理层（老板、区域经理、运营总监），你关注**全局经营状况**——哪家门店执行到位、哪家顾客情绪异常、员工在关注什么问题。Lingtin 帮你用数据代替「走店凭感觉」。

### 总部 vs 区域管理层

- **总部管理层**（老板）：看到**所有门店**的数据
- **区域管理层**（区域经理）：只看到**管辖门店**的数据，额外显示「基准对比」（我的区域 vs 全公司）和「改善亮点」

系统根据你的账号自动判断权限范围，无需手动切换。如果你看到页面标题旁有「管理 N 家门店」标签，说明你是区域管理层。

## 四个核心页面

| 页面 | 路径 | 一句话说明 |
|------|------|-----------|
| **AI 智库** | `/admin/chat` | 登录后首页——每日智能汇报 + 跨门店数据分析 + 行动链接 |
| **总览** | `/admin/briefing` | 紧凑指标行 + 问题卡片 + 今日关键词 + 门店网格 |
| **洞察** | `/admin/insights` | 顾客洞察 / 产品洞察 / 员工反馈，分段切换 |
| **会议** | `/admin/meetings` | 跨门店会议纪要浏览 + 录制经营会 / 店长沟通 |

> 问卷管理入口：点击页面右上角头像 → 选择「问卷管理」（路径：`/admin/question-templates/manage`）。
> 区域管理入口：点击页面右上角头像 → 选择「区域管理」（路径：`/admin/regions`）。
> 所有角色均可通过头像菜单进入「提交反馈」和「我的反馈」。

## 典型使用场景

```
──── 早间（2分钟）────
1. 打开 App → AI 智库自动推送每日汇报（异常门店、重点问题、行动建议）
2. 点击汇报中的行动链接按钮，直接跳转到总览或洞察页

──── 深入巡查（5分钟）────
3. 切到总览页，查看指标行（门店数/桌访数/覆盖率/情绪分）
4. 查看问题卡片，点击展开看顾客原话 + Q&A 对话，听原声录音
5. 门店网格快速对比各店表现（支持日期范围：昨日/前天/近7天/近30天）

──── 深入分析（按需）────
6. 切到「洞察」→ 顾客标签查看近 7 天建议趋势和反馈热词
7. 切到「洞察」→ 产品标签看员工关注话题，发现跨门店共同问题
8. 切到「洞察」→ 员工反馈标签查看/处理团队提交的产品反馈

──── 经营会 / 店长沟通 ────
9. 切到「会议」→ 点击「录制会议」
10. 默认录制经营会（全部门店），或下拉选择门店录制店长沟通
11. 录音完成后查看 AI 会议纪要（摘要 + 关键决定 + 行动待办）

──── 周度管理 ────
12. AI 智库提问：本周趋势、跨门店对比
13. 从头像菜单进入问卷管理，更新标准问题
```

## 总览（登录后首页）

合并了原简报和经营看板，提供统一入口。

### 紧凑指标行

顶部一行四个核心数字：门店总数、今日桌访总数、平均覆盖率、平均情绪分。快速掌握全局状态。

> **区域管理层**：指标只统计管辖门店。

### 基准对比（区域管理层专属）

总览页下半部分会显示：

- **基准对比面板** — 情绪分 / 覆盖率 / 待办完成率，我的区域 vs 全公司。绿色=领先，红色=落后
- **智能信号** — 系统自动检测执行缺口（连续无桌访、待办超期）和趋势异常（情绪连续下降）
- **改善亮点** — 全公司最佳门店展示，自己门店标绿色

### 问题卡片

系统自动扫描所有门店数据，检测异常并生成问题清单。

**异常检测规则：**
- 红色圆点（严重）：菜品集中差评（同菜 ≥2 桌）、服务速度投诉（≥2 桌）、员工态度问题、情绪异常（< 0.5）
- 黄色圆点（关注）：环境问题、覆盖率下降（< 70%）、桌访未执行、高优行动建议超 3 天未处理

**每个问题卡片包含：**
- 门店名称 + 问题类型 + 严重度圆点
- 顾客原话（最多 3 条）

**溯源操作：**
- 点击卡片展开 → 查看完整 Q&A 对话（店长提问 + 顾客回答）
- 点击播放按钮 → 听原始桌访录音

**空状态：** 所有门店正常时显示「一切正常」+ 平均情绪/覆盖率统计。

### 今日关键词

跨门店顾客反馈关键词汇聚，词越大提及越多。快速了解今天顾客最关注什么。

### 门店网格

每家门店一张卡片，显示桌访数、覆盖率、平均情绪分。点击进入门店详情页，查看该店的桌访记录和反馈。

门店详情页内容：
- 每条记录显示桌号、时间、情绪标签（绿/灰/红）、AI 摘要
- 点击展开查看：转写全文、反馈详情、菜品提及

## 洞察

合并了顾客洞察和产品洞察，通过顶部分段控制切换。

### 顾客标签

两个核心区块，帮你从顾客视角发现机会和问题。

**顾客建议（近 7 天）：** 跨门店聚合顾客的建设性建议（如"建议加个冷面"、"希望有儿童套餐"）。AI 自动从桌访反馈中识别「建议」类型，区别于差评投诉。
- 按建议文本聚合，显示提及次数和来源门店
- 点击展开 → 查看所有来源桌号 + Q&A 对话详情 + 播放原声录音

**反馈热词：** 高频差评和好评关键词分区展示。
- 差评词和好评词分别排列，大小表示频率
- 点击热词展开查看相关对话详情 + 原声播放

### 产品标签

跨门店员工问题聚类分析，了解团队在关注什么。
- 基于员工 AI 智库对话 + 桌访内容，自动归类为 7+1 个话题
- 话题卡片按关注人数排序
- 3 家以上门店关注同一话题时标记「共同关注」标签

### 员工反馈标签

管理层查看所有角色提交的产品反馈，统一管理。

**反馈卡片：** 按提交时间倒序排列，每条显示：
- 提交者姓名、反馈分类（系统故障/功能需求/易用性等）、优先级标签
- AI 自动生成的摘要
- 反馈详细内容 + 标签
- 语音反馈可点击播放原声

**筛选：** 顶部两个下拉框，按状态或分类过滤。

**操作：**
- **更新状态** — 下拉选择：待处理 → 已读 → 处理中 → 已解决 / 已关闭
- **回复反馈** — 点击「回复」输入回复内容，提交者在「我的反馈」页可看到回复

## 提交产品反馈

管理层也可以提交自己的产品反馈和需求（功能入口与所有角色相同）。

**入口：** 右上角头像 → 「提交反馈」（路径：`/feedback`）。

**提交方式：**
- **文字模式**：输入反馈内容，可附带最多 3 张图片
- **语音模式**：录音后自动转写并分类

**查看我的反馈：** 头像 → 「我的反馈」（路径：`/feedback/history`），查看提交历史、处理状态和回复。

## 会议

管理层专属的会议录制和浏览功能，支持两种会议类型。

### 会议概览页

路径：`/admin/meetings`。按日期浏览各门店的会议纪要，包括：
- **各门店会议**：按门店分组展示会议记录，点击查看 AI 纪要
- **我的会议**：管理层自己录制的会议，折叠展示

页面顶部有「录制会议」按钮进入录制页。

### 会议录制

路径：`/admin/meetings/record`。全屏录制模式。

**会议类型通过「关联门店」自动推导：**
- **经营会（默认）**：不选门店 → 跨门店经营分析会。AI 侧重跨店对比和统一决策
- **店长沟通**：下拉选择具体门店 → 与该店店长的一对一沟通。AI 侧重根因分析和承诺事项

**操作步骤：**
1. 点击右上角下拉按钮，默认为「经营会 · 全部门店」
2. 如需录制店长沟通，下拉选择目标门店
3. 参考「智能议题」卡片（基于昨日简报数据）确定讨论重点
4. 点击录音按钮开始，录音中下拉不可切换
5. 停止后自动上传 + AI 分析，生成会议纪要（摘要 + 关键决定 + 行动待办）

## 问卷管理

从页面右上角头像菜单进入「问卷管理」。配置桌访标准问题模板，统一指导店长「今天重点问什么」。

### 核心操作

- **查看 / 新建 / 编辑 / 删除模板**
- 每个模板包含模板名称、问题条目列表、生效日期、启用状态
- 保存后立即对店长生效

### 最佳实践

- 问题控制在 3-5 个
- 定期根据数据反馈调整
- 新菜上市时必配问卷

## AI 智库（登录后首页）

### 每日智能汇报

每天首次打开 App，AI 自动推送一份管理层视角的智能汇报，内容包括：
- 异常门店检测（覆盖率低、情绪异常、集中差评）
- 跨门店重点问题
- 行动建议和待处理事项

汇报中的**行动链接**渲染为可点击的按钮（如「查看总览 →」「查看洞察 →」），点击直接跳转到对应页面。汇报末尾的**追问建议**显示为快捷 pill，点击即可继续提问。

> 每日汇报每天只推送一次。清空对话后会重新触发。
> 区域管理层的汇报自动限定为管辖门店范围。

### 跨门店数据分析

数据范围覆盖所有门店（区域管理层限定为管辖门店），可进行跨门店对比分析。

**管理层专属问题示例：**

| 场景 | 示例 |
|------|------|
| 门店对比 | 哪家门店本周覆盖率最高/最低？ |
| 全局趋势 | 所有门店这周平均情绪分比上周有变化吗？ |
| 菜品分析 | 哪道菜在多家门店同时收到差评？ |
| 员工表现 | 哪位店长的桌访覆盖率最高？ |

## 常见问题

- **问卷管理入口在哪？** → 右上角头像菜单 → 问卷管理。
- **总览页没有问题卡片？** → 说明所有门店运营正常，会显示「一切正常」统计。
- **原来的经营看板去哪了？** → 已合并到总览页，门店网格和指标行提供相同信息。
- **怎么看单个门店详情？** → 总览页门店网格点击任意门店卡片进入。
- **经营会和店长沟通有什么区别？** → 经营会不关联门店，AI 侧重跨店对比；店长沟通关联具体门店，AI 侧重根因分析。
- **顾客洞察和产品洞察去哪了？** → 已合并为「洞察」页，通过顶部标签切换。
- **员工反馈在哪看？** → 洞察页切换到「员工反馈」标签。
- **怎么提交反馈？** → 右上角头像 → 提交反馈。支持文字和语音两种方式。
- **区域管理入口在哪？** → 右上角头像菜单 → 区域管理。可创建区域、分配门店和管理员。
- **新开了一家门店，区域管理员需要手动添加吗？** → 不需要。只要把新门店挂到区域下，该区域的管理员自动就能看到新店数据。
- **区域管理和直接配 managed_restaurant_ids 有什么区别？** → 区域管理是"按区域分组"的快捷方式，新门店加入区域后自动继承权限；直接配 ID 列表优先级更高但需手动维护。
