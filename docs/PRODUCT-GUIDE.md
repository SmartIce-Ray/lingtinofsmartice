## 产品简介

**Lingtin** 是餐饮行业的语音驱动管理工具，帮助团队建立闭环：**说了 → 记了 → 做了 → 验了**。

通过录音采集顾客真实反馈，由 AI 自动分析并生成可执行的改善建议，让每一次桌访都转化为门店进步的动力。

### 核心角色

| 角色 | 适用人群 | 登录后跳转 | 可访问页面 |
|------|----------|-----------|-----------|
| **店长** | 店长、值班经理 | `/recorder` 录音页 | 录音、看板、AI 智库 |
| **管理员** | 老板、区域经理 | `/admin/briefing` 总览 | 总览、洞察、会议、AI 智库 |
| **厨师长** | 厨师长、主厨 | `/chef/dashboard` 厨房待办 | 厨房待办、菜品反馈、厨房会议 |

## 访问与登录

### 访问地址

在浏览器中打开 **https://lt.smartice.ai**，推荐使用手机浏览器（iOS Safari / Android Chrome）。Lingtin 是 PWA 应用，可添加到手机主屏幕获得接近原生 App 的体验。

### 登录步骤

1. 打开访问地址，系统自动跳转到登录页面
2. 输入**用户名**和**密码**（由管理员提供）
3. 点击「登录」
4. 系统根据角色自动跳转到对应页面

> 登录状态保存在浏览器中，关闭后再次打开无需重新登录。切换账号需清除浏览器缓存。

### 首次使用提醒

- **店长**：首次录音时浏览器会弹出麦克风授权请求，请点击「允许」。误点「拒绝」需到浏览器设置中手动开启。
- **厨师长**：菜品反馈数据来源于店长的桌访录音。如果菜品页没有数据，请确认店长当日已完成桌访。
- **所有角色**：建议添加到主屏幕（iOS: Safari → 分享 → 添加到主屏幕；Android: Chrome → 菜单 → 添加到主屏幕）。

## 店长使用指南

> 独立文档：[docs/user-guides/store-manager.md](user-guides/store-manager.md)

作为店长，你是门店与顾客之间最直接的桥梁。Lingtin 帮助你把每天的桌访从「完成任务」变成「获取洞察」——通过 AI 分析真实的顾客声音，让你清楚地知道哪些做得好、哪些需要改进。

### 三个核心页面

| 页面 | 路径 | 说明 |
|------|------|------|
| **录音** | `/recorder` | 桌访录音 + 例会录音，AI 自动转写分析 |
| **看板** | `/dashboard` | 今日数据一览——覆盖率、情绪、顾客反馈、话术评估、行动建议 |
| **AI 智库** | `/chat` | 向 AI 提问，获取经营洞察和建议 |

### 典型一天的流程

```
──── 午市前 ────
1. 打开录音页 → 切到「例会」→ 选择「餐前会」
2. 查看"昨日复盘待办"提醒卡片，了解今天需要跟进的事项
3. 录下餐前会，会后查看 AI 纪要

──── 午市中 ────
4. 切回「桌访」→ 选桌号 → 录音
5. 每桌录完后查看列表确认状态

──── 午市后 ────
6. 打开看板页，查看覆盖率和情绪概览
7. 处理 AI 行动建议

──── 晚市 ────
8. 重复桌访录音流程

──── 收餐后（21:00 前后）────
9. 系统自动生成"当日总结 + 复盘议题"
10. 切到「例会」→ 选择「每日复盘」
11. 查看"今日议题"卡片，按议题顺序开复盘会
12. 录音完成后，AI 自动提取任务 → 写入行动建议
    （次日餐前会时自动展示为"昨日待办"）
```

### 桌访录音

#### 为什么录音

手写记录容易遗漏细节，事后回忆更不可靠。录音让 AI 捕捉到顾客的每一句话，你只需专注和顾客交流。

#### 累计统计

录音页顶部显示三项累计数据：
- **X 次桌访**：累计完成的桌访录音数
- **X 位好评**：累计获得的正面评价数
- **X 已改善**：累计解决的行动建议数

开始录音后自动隐藏，不影响操作。

#### 操作步骤

1. **选择桌号** — 确保顶部标签在「桌访」，点击桌号选择器。必须先选桌号才能录音。
2. **查看问卷提示（如有）** — 管理层配置的标准问题会显示在录音按钮上方，点击可展开/折叠。
3. **开始录音** — 点击红色录音按钮，出现波形动画和计时器。建议单条 1 分钟内，手机距顾客 30-50cm。
4. **停止录音** — 再次点击按钮停止。后台处理：上传(~2秒) → 语音识别(~3-5秒) → AI 分析(~7-10秒) → 完成。
5. **查看录音历史** — 录音按钮下方显示今日/昨日列表。状态：已保存→上传中→处理中→已完成/失败（可重试）。

#### 离线支持

网络不好时录音先保存在本地（最多 20 条），恢复网络后自动上传。

### 例会录音

#### 会议类型

| 会议类型 | 适用场景 | 典型时长 |
|----------|----------|----------|
| **餐前会** | 每天开餐前——注意事项、推荐菜品、人员分工 | 5-10 分钟 |
| **每日复盘** | 收餐后回顾——总结问题、表扬亮点、安排次日 | 10-15 分钟 |
| **周例会** | 每周综合会议——回顾本周、分析趋势、制定计划 | 20-30 分钟 |
| **厨房会议** | 厨房内部会议——备餐讨论、品质复盘 | 5-15 分钟 |

> 选择正确的会议类型很重要——AI 根据不同类型采用不同分析策略。

#### 操作步骤

1. **切换到例会标签** — 录音页顶部点击「例会」
2. **选择会议类型** — 不同类型配色不同（餐前会蓝/复盘琥珀/周例会紫/厨房会议绿）
3. **查看上下文卡片** — 每日复盘显示今日议题；餐前会显示昨日待办
4. **录音** — 手机放在会议桌中央，录完后停止
5. **查看 AI 纪要** — 会议摘要 + 关键决定 + 行动待办。复盘会和餐前会的行动待办**自动写入行动建议系统**

### 每日运营闭环

连接桌访、复盘会和餐前会的自动化数据流转：

```
白天桌访 → 21:00 自动生成今日总结 + 复盘议题
    ↓
复盘会：按议题讨论 → AI 提取任务 → 自动入库
    ↓
次日餐前会：自动展示昨日待办 → 提醒跟进
```

- **复盘会议题**：21:00 自动聚合当日桌访，按严重度排列（红/黄/蓝）
- **餐前会待办**：自动查询昨日 pending 任务，显示内容、负责人和截止时间

### 数据看板

六个核心区块：

1. **执行覆盖率** — 午市/晚市桌访执行情况（≥90% 正常、70-90% 差X桌、<70% 红色）
2. **情绪概览** — 正面/中性/负面分布 + 昨日对比趋势箭头（绿↑/红↓）。点击反馈气泡查看原始对话，有音频时可点播放按钮听原声
3. **顾客反馈** — 全维度反馈（菜品/服务/态度/环境），按严重度排序，每项附顾客原话。可展开查看 Q&A 对话详情 + 播放原声
4. **顾客建议** — 近 7 天顾客提出的建设性建议（如"建议加个冷面"），AI 自动从反馈中识别并聚合
5. **话术使用** — 桌访中的提问分优劣展示（优秀示范 / 可以更好），各附简短原因。「AI 优化」按钮跳转智库获取优化建议
6. **AI 行动建议** — 详见下节

### AI 行动建议

- **生成**：点击「生成今日建议」→ AI 分析负面反馈 → 3-5 条建议（10-20 秒）
- **优先级**：高（红，3+桌）、中（黄，2桌）、低（蓝，1桌）
- **分类**：菜品质量 / 服务速度 / 环境 / 员工态度 / 其他
- **操作**：知悉（已了解）、已解决（可附备注）、忽略（偶发不处理）
- 复盘会产生的任务也自动出现在这里

### AI 智库

通过底部导航进入「智库」，用自然语言提问。空状态时系统根据当天覆盖率和负面反馈动态生成个性化快捷问题。

| 场景 | 示例 |
|------|------|
| 日常复盘 | 今天顾客整体满意度怎么样？ |
| 菜品分析 | 本周被差评最多的菜是哪道？ |
| 话术改进 | 帮我分析最近的桌访话术，怎么优化？ |
| 趋势追踪 | 这周和上周相比，覆盖率有变化吗？ |
| 问题排查 | A3 桌顾客说了什么？ |

> AI 只能访问**本店数据**，其他门店看不到你的信息。

### 提交产品反馈

所有角色都可以随时提交产品反馈和需求，帮助改进 Lingtin 系统。

**入口：** 点击页面右上角头像 → 选择「提交反馈」（路径：`/feedback`）。

**提交方式：**
- **文字模式**：输入反馈内容，可附带最多 3 张图片（点击输入框右下角相机按钮添加）
- **语音模式**：点击录音按钮说出反馈，录音完成后上传，系统自动转写并分类
- 两种模式都支持添加图片附件

**AI 自动分类：** 提交后 AI 自动识别反馈类型（系统故障/功能需求/易用性/性能/内容质量/其他）、优先级和摘要。

**查看我的反馈：** 头像菜单 → 「我的反馈」（路径：`/feedback/history`），可查看提交历史、处理状态和管理层回复。

### 店长常见问题

- **录音按钮点不了？** → 检查是否已选择桌号。
- **一直显示处理中？** → 正常 10-15 秒。超过 1 分钟刷新页面。失败后点「重试」。
- **顾客说方言能识别吗？** → 可以，支持 200 多种方言自动识别。
- **例会和桌访有什么区别？** → 桌访分析顾客情绪和菜品反馈；例会生成摘要、关键决定和行动待办。

## 管理层使用指南

> 独立文档：[docs/user-guides/management.md](user-guides/management.md)

作为管理层（老板、区域经理、运营总监），你关注**全局经营状况**——哪家门店执行到位、哪家顾客情绪异常、员工在关注什么问题。Lingtin 帮你用数据代替「走店凭感觉」。

### 四个核心页面

| 页面 | 路径 | 说明 |
|------|------|------|
| **总览** | `/admin/briefing` | 登录后首页——紧凑指标行 + 问题卡片 + 今日关键词 + 门店网格 |
| **洞察** | `/admin/insights` | 顾客洞察 / 产品洞察 / 员工反馈，分段切换 |
| **会议** | `/admin/meetings` | 跨门店会议纪要浏览 + 录制经营会 / 店长沟通 |
| **AI 智库** | `/admin/chat` | 跨门店数据分析，自然语言提问 |

> 问卷管理从头像菜单进入（路径：`/admin/question-templates/manage`）。
> 所有角色均可通过头像菜单进入「提交反馈」和「我的反馈」。

### 典型使用场景

```
──── 早间巡查（5分钟）────
1. 打开总览页，一眼看到指标行（门店数/桌访数/覆盖率/情绪分）
2. 查看问题卡片——哪些门店有异常，点击展开看顾客原话和 Q&A 对话
3. 浏览今日关键词云，了解顾客高频反馈
4. 门店网格快速对比各店表现

──── 深入分析（按需）────
5. 切到「洞察」→ 顾客标签查看建议趋势和反馈热词
6. 切到「洞察」→ 产品标签看员工关注话题
7. 切到「洞察」→ 员工反馈标签查看/处理团队提交的产品反馈

──── 经营会 / 店长沟通 ────
8. 切到「会议」→ 点击「录制会议」
9. 默认录制经营会，或下拉选择门店录制店长沟通
10. 录音完成后查看 AI 会议纪要

──── 周度管理 ────
11. AI 智库提问：本周趋势、跨门店对比
12. 从头像菜单进入问卷管理，更新标准问题
```

### 总览（登录后首页）

合并了原简报和经营看板，提供统一入口：

**紧凑指标行：** 门店总数、今日桌访总数、平均覆盖率、平均情绪分，一行四数快速掌握全局。

**问题卡片：** 系统自动扫描所有门店数据，检测异常并生成问题清单。
- 严重度用彩色圆点区分（红色=严重、黄色=关注）
- 每张卡片包含门店名称 + 问题类型 + 顾客原话
- 点击卡片展开查看完整 Q&A 对话（店长提问 + 顾客回答）+ 播放原声录音
- 空状态时显示「一切正常」+ 整体指标统计

**今日关键词：** 跨门店顾客反馈关键词汇聚，词越大提及越多。

**门店网格：** 每家门店显示桌访数、覆盖率、平均情绪分，点击进入门店详情。

### 洞察

合并了顾客洞察和产品洞察，通过顶部分段控制切换。

**顾客标签：**
- 顾客建议（近 7 天）：跨门店聚合建设性建议，按提及次数排序，点击展开查看来源桌号 + Q&A 对话 + 原声播放
- 反馈热词：高频差评和好评关键词分区展示，点击热词展开查看相关对话

**产品标签：**
- 跨门店员工问题聚类，自动归类为 7+1 个话题，按关注人数排序
- 3 家以上门店关注同一话题时标记为「共同关注」

**员工反馈标签：**
- 管理层查看所有角色提交的产品反馈，按时间倒序排列
- 支持按状态（待处理/已读/处理中/已解决/已关闭）和分类（系统故障/功能需求/易用性/性能/内容质量/其他）筛选
- 每条反馈显示提交者、分类、优先级、AI 摘要、内容详情、标签
- 操作：下拉更新状态、回复反馈（提交者可在"我的反馈"页看到回复）
- 语音反馈可点击播放原声

### 会议

管理层专属的会议录制和浏览功能。

**会议概览** (`/admin/meetings`)：按日期浏览各门店会议纪要 + 管理层自己录制的会议。

**会议录制** (`/admin/meetings/record`)：全屏录制模式，通过「关联门店」下拉自动推导会议类型：
- 不选门店 = **经营会**（跨门店经营分析），AI 侧重跨店对比和统一决策
- 选择门店 = **店长沟通**（一对一沟通），AI 侧重根因分析和承诺事项
- 智能议题卡片辅助讨论（基于昨日简报数据）
- 录音完成后自动生成 AI 纪要（摘要 + 关键决定 + 行动待办）

### 门店详情

从总览页点击门店卡片进入。展示该门店的桌访记录列表：
- 每条记录显示桌号、时间、情绪标签（绿/灰/红）、AI 摘要
- 点击展开查看：转写全文、反馈详情、菜品提及

### 问卷管理

从页面右上角头像菜单进入「问卷管理」。配置桌访标准问题模板，统一指导店长「今天重点问什么」。

**核心操作：** 查看 / 新建 / 编辑 / 删除模板。每个模板包含模板名称、问题条目列表、生效日期、启用状态。

**最佳实践：** 问题控制在 3-5 个，定期根据数据反馈调整，新菜上市时必配问卷。

### AI 智库（管理层版）

与店长版功能相同，但**数据范围覆盖所有门店**，可进行跨门店对比分析。

| 场景 | 示例 |
|------|------|
| 门店对比 | 哪家门店本周覆盖率最高/最低？ |
| 全局趋势 | 所有门店这周平均情绪分比上周有变化吗？ |
| 菜品分析 | 哪道菜在多家门店同时收到差评？ |
| 员工表现 | 哪位店长的桌访覆盖率最高？ |

## 厨师长使用指南

> 独立文档：[docs/user-guides/chef.md](user-guides/chef.md)

作为厨师长，你是后厨品质的最终把关人。Lingtin 把顾客对菜品的真实评价直接送到你面前——不用等店长口头转述，3 秒看到哪些菜需要改进、顾客原话怎么说的。

### 三个核心页面

| 页面 | 路径 | 说明 |
|------|------|------|
| **厨房待办** | `/chef/dashboard` | 与厨房相关的改善任务，按优先级排序，高频问题置顶高亮 |
| **厨房反馈** | `/chef/dishes` | 全维度反馈（口味/速度/温度/卖相/食材）+ 对话溯源 + 播放录音 |
| **厨房会议** | `/chef/meetings` | 厨房内部会议录音，AI 自动生成纪要 |

### 典型一天的流程

```
──── 午市前 ────
1. 打开「厨房待办」→ 查看昨日差评产生的改善任务
2. 重点关注置顶高亮的高频问题，安排当日改进

──── 午市/晚市后 ────
3. 切到「厨房反馈」→ 查看差评菜
4. 展开看顾客原话 → 不确定时点播放按钮听录音
5. 切换「今日/昨日」对比变化

──── 收餐后 ────
6. 参加复盘会，或在「厨房会议」查看 AI 会议纪要
7. 根据反馈调整次日备餐计划
```

### 厨房反馈（核心功能）

**功能价值：**
- 差评菜品优先展示，3 秒定位问题
- 全维度覆盖：不只口味，还有出菜速度、温度、卖相、食材新鲜度
- 直接看顾客原话（"偏咸" ×2），不是二手转述
- 不确定时点播放按钮听录音，完全还原现场

**操作步骤：**

1. **进入厨房反馈** — 底部导航点「菜品」，页面分为上下两个区域
2. **查看需要关注的菜品** — 上方「需要关注」区展示有差评的菜品，按差评数从多到少排列。每道菜显示菜名 + 差评桌数 + 反馈标签（如「偏咸 ×2」）
3. **展开查看对话详情** — 点击菜品卡片展开，看到每桌的完整对话（桌号 + 问题标签 + 店长提问 + 顾客回答）
4. **播放原声录音** — 对话区点击播放按钮听原始录音。同时只能播放一条
5. **执行行动** — 每项问题可点「已改善」标记处理或「查看待办」跳转待办页
6. **切换日期** — 顶部切换「今日/昨日」对比变化
7. **查看好评菜品** — 下方「表现不错」区展示零差评菜品

### 厨房待办

只看与厨房相关的待办（菜品质量 + 出菜速度等），不被服务、环境等无关建议干扰。

- 任务按优先级排列，「建议优先处理」区域置顶高亮高频 + 高优先级问题
- 操作：知悉 / 已解决（可附备注） / 忽略

### 厨房会议

厨房晨会、备餐讨论也能 AI 记录，自动生成纪要。

操作：底部导航点「会议」→ 选择「厨房会议」类型 → 录音 → 查看 AI 纪要（摘要 + 关键决定 + 行动待办）。录音建议 5-15 分钟，手机放在讨论区域中央。

### 提交产品反馈

厨师长也可以通过头像菜单提交产品反馈，操作方式与店长相同（详见[店长 > 提交产品反馈](#提交产品反馈)）。

### 厨师长常见问题

- **差评原话不准确？** → 点播放按钮听录音确认，AI 转写偶尔有偏差。
- **没有菜品数据？** → 菜品反馈来源于店长的桌访录音，需店长先完成当日桌访。
- **好评菜也想看详情？** → 目前好评区只显示汇总，详细对话后续支持。

## 术语表

### 核心概念

| 术语 | 解释 |
|------|------|
| **桌访** | 店长主动走到顾客桌前交流并录音收集反馈的行为，是系统的核心数据来源 |
| **覆盖率** | 当天桌访数 / 开台数。如午市开 20 桌、走访 15 桌，覆盖率 75% |
| **情绪分** | AI 对顾客情绪的量化评估（0.00~1.00）。≥0.80 正面（绿）、0.40~0.79 中性（灰）、<0.40 负面（红） |
| **行动建议** | AI 根据负面反馈自动生成的改善建议，含优先级、分类和具体措施 |
| **问卷提示** | 管理层配置的标准问题清单，录音时显示在店长手机上，提醒「今天应该问什么」 |
| **每日总结** | 系统每晚 21:00 自动聚合当日桌访数据生成的结构化议题，供复盘会使用 |
| **顾客建议** | AI 从桌访反馈中自动识别的建设性建议（区别于差评投诉），如"建议加个冷面" |

### 录音相关

| 术语 | 解释 |
|------|------|
| **STT** | Speech-to-Text，语音转文字。使用 DashScope Paraformer-v2（讯飞回退），支持普通话和方言 |
| **菜名纠偏** | AI 将口语/方言菜名（如"清蒸路鱼"）自动纠正为标准菜名（"清蒸鲈鱼"） |

### 会议相关

| 术语 | 解释 |
|------|------|
| **餐前会** | 每天开餐前的简短会议，内容包括注意事项、推荐菜品、人员分工 |
| **每日复盘** | 收餐后回顾当天运营的会议，总结问题、表扬亮点、安排次日工作 |
| **周例会** | 每周一次综合会议，回顾本周表现、分析趋势、制定下周计划 |
| **厨房会议** | 厨房内部会议——备餐讨论、品质复盘，AI 自动生成纪要 |
| **AI 会议纪要** | AI 对例会录音的分析结果：会议摘要 + 关键决定 + 行动待办 |
| **复盘议题** | 系统基于当日桌访数据 AI 生成的议题卡片，按严重度排序，在复盘会录音前展示 |
| **经营会** | 管理层跨门店经营分析会议，AI 侧重跨店对比和统一决策 |
| **店长沟通** | 管理层与单店店长的一对一沟通，AI 侧重根因分析和承诺事项 |

### 看板相关

| 术语 | 解释 |
|------|------|
| **午市/晚市** | 一天中的两个用餐时段，系统按时段分别统计 |
| **菜品提及** | 桌访中顾客主动提到的菜品，AI 自动统计次数和好差评 |
| **话术红黑榜** | AI 从桌访中提取的店长话术，分优秀示范和待改进两类 |
